# 安心睡策略組合

兩支互補的策略，目標都是「睡覺時不會被嚇醒」。

## 1. `TXF_早盤當沖_安心睡.xs` —— 主動賺價差

**標的**：台指期 (TXF)
**週期**：1 分 K
**進場窗口**：09:00 – 10:30
**強制平倉**：11:30

### 為什麼這樣設計
- 台股 09:00–10:30 是動能最強、訊號最乾淨的時段
- 11:30 一律出場 → 午餐前空手、下午、夜盤、結算日通通不關你的事
- 三層出場（硬停損 / 保本 / 跟蹤鎖利）全自動，不需要你看盤

### 核心出場規則
| 觸發 | 動作 |
|---|---|
| 進場價 -1.5×ATR | 硬停損全出 |
| 高點達到 +0.5×ATR | 啟動保本（回成本價就出） |
| 高點達到 +1.0×ATR | 跟蹤鎖利（高點回落 1×ATR 出） |
| 時間到 11:30 | 一律全出 |

### 參數調整方向
- 想更保守 → `ATRMultSL` 從 1.5 降到 1.0
- 想拉長持倉 → `ForceExit` 從 113000 改 130000（但會吃到午盤盤整）
- 訊號太少 → 把 `condition41` 的 0.25 放寬到 0.5

## 2. `0050_現股網格_睡眠版.xs` —— 被動賺波動

**標的**：0050 / 006208
**週期**：日 K
**核心**：底倉 + 等距網格 + 熊市保護

### 為什麼這樣設計
- ETF 現股不會歸零（最壞情況是套牢，不會爆倉）
- 網格賺的是「波動費」，不需要預測方向
- 熊市自動停止加倉 → 不會在跌勢中無腦攤平到破產
- 回撤超過 30% 自動減倉 → 強制承認錯誤、保留現金

### 核心規則
| 情境 | 動作 |
|---|---|
| 第一次跑 | 建立 3 張底倉 |
| 跌 3%（且不在熊市） | 加 1 格 |
| 漲 3% | 賣 1 格（保留底倉不動） |
| 跌破 200 日線 | 停止加倉（保命） |
| 回撤 -30% | 賣掉加倉部位的一半 |
| 總獲利 +25% | 賣掉加倉部位的 1/3 |

### 參數調整方向
- 想抓更小波動 → `GridPct` 從 3.0 降到 2.0（但會更頻繁交易、手續費增加）
- 資金多 → 提高 `MaxGrids` 和 `EachGridSize`
- 想更安全 → `DrawdownPct` 從 -30 改 -20（提早減倉）

---

## 為什麼不在台指期做網格
- 期貨有結算日，網格無法持續累積
- 高槓桿 + 留倉夜盤 = 跳空爆倉風險
- 趨勢盤一單邊行情，反向倉位直接被掃光
- 違背「安心睡」的設計初衷

如果你之後想做加密貨幣網格（BTC/ETH 現貨），再另外開一份。
