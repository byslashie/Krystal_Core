# 元大庫存同步系統說明

> 最後更新：2026-04-14

---

## 一、整體流程圖

```
元大 OneAPI（Windows DLL）
        │
        ▼  32-bit Python（.venv_yuanta32_new）
sync_yuanta_positions.py
  ├─ 登入 API（約 40 秒等待）
  ├─ 發送 RQ 0015（庫存查詢）
  ├─ 等待 OnResponse（約 15 秒）
  ├─ 解析回應 → 結構化資料
  ├─ 存檔 → data/yuanta_positions_snapshot.json  ◀─ 關鍵中繼檔
  └─ 嘗試直寫 Sheets（32-bit SSL 可能失敗，無妨）

        │  JSON snapshot
        ▼  64-bit Python（系統 Python）
upload_yuanta_to_sheets.py
  ├─ 讀取 snapshot.json
  ├─ 刪除 broker_positions 中舊的元大行（避免重複）
  ├─ 寫入新資料（9 欄位）
  ├─ 記錄 sync_logs
  └─ 發送 Discord 通知

        │  broker_positions 已更新
        ▼  64-bit Python（排程 13:40）
record_daily_nav.py
  ├─ 讀 snapshot.json（元大市值）
  ├─ 讀 broker_positions（IB 市值）
  └─ Upsert → portfolio_daily（一天一筆）
```

## 二、Windows 排程（已設定，自動執行）

| 排程名稱 | 執行時間 | 執行對象 | 說明 |
|----------|----------|----------|------|
| Krystal-元大同步-0915 | **週一～五 09:45** | `sync_yuanta_09h15.bat` | 開盤後同步庫存 |
| Krystal-元大同步-1230 | **週一～五 12:30** | `sync_yuanta_09h15.bat` | 盤中更新 |
| Krystal-元大同步-1335 | **週一～五 13:35** | `sync_yuanta_09h15.bat` | 收盤後同步 |
| Krystal-每日後記錄-1340 | **週一～五 13:40** | `record_daily_nav.bat` | 記錄每日總資產 |

> 排程由 Windows 工作排程器管理，電腦必須**開機且未鎖定使用者**才會執行。

### sync_yuanta_09h15.bat 執行順序

```
Step 1  .venv_yuanta32_new\python.exe brokers\sync_yuanta_positions.py
         ├─ 登入 YuantaOneAPI（需憑證，約 1～2 分鐘）
         ├─ 查詢庫存並存 JSON snapshot
         └─ 若失敗 → 跳過 Step 1，直接用現有 snapshot

Step 2  python.exe brokers\upload_yuanta_to_sheets.py
         └─ 從 snapshot.json 上傳至 Google Sheets
```

## 三、投資組合每日記錄排程

> 依賴元大 13:35 同步完成後，才能拿到最新市值，因此設定在 **13:40**。

```
每日 13:35  Krystal-元大同步-1335
              └─ 更新 broker_positions（含元大最新現價、市值）

每日 13:40  Krystal-每日後記錄-1340
              └─ record_daily_nav.py
                   ├─ 讀 data/yuanta_positions_snapshot.json → 元大市值
                   ├─ 讀 broker_positions → IB 市值
                   └─ 寫入 portfolio_daily（一天一筆，重複執行會更新當日）
```

**portfolio_daily 每日新增一列：**

| 欄位 | 說明 |
|------|------|
| `date` | 日期（YYYY-MM-DD） |
| `yuanta_market_value` | 元大總市值（NTD） |
| `yuanta_unrealized_pnl` | 元大未實現損益 |
| `ib_market_value` | IB 總市值 |
| `ib_unrealized_pnl` | IB 未實現損益 |
| `total_market_value` | 全帳戶合計市值 |
| `total_unrealized_pnl` | 全帳戶合計未實現損益 |

**Dashboard 圖表（📊 Portfolio 頁，Drawdown 圖下方）：**

| 圖 | 來源欄位 |
|----|---------|
| 💰 每日總資產趨勢 | `total_market_value` |
| 🏦 各券商市值堆疊 | `yuanta_market_value` / `ib_market_value` |
| 🥧 今日佔比 Pie | 最新一筆各券商比例 |

## 八、常見問題排解

### ❌ 排程執行但庫存沒更新

1. 查看 `logs\yuanta_sync_YYYYMMDD_HHMMSS.log`
2. 確認元大憑證是否過期（RtnCode=141 → 重新安裝憑證）
3. 確認 `.env` 中帳號密碼正確

### ❌ RtnCode=141（憑證不存在）

1. 找到 `.pfx` 或 `.p12` 憑證檔（向元大申請）
2. 雙擊安裝 → 選「目前使用者」→ 輸入密碼 → 自動選擇存放區
3. 重新執行 Step 1

### ❌ Step 1 失敗，Step 2 用舊 snapshot

- 這是正常 fallback 行為
- Dashboard 顯示的是 snapshot.json 的時間戳，可確認資料新鮮度
- 手動執行 Step 1 即可更新 snapshot

### ❌ portfolio_daily 分頁不存在

1. 在 Google Sheets 手動新增分頁，命名為 `portfolio_daily`
2. 執行 `record_daily_nav.py` 會自動建立 header
