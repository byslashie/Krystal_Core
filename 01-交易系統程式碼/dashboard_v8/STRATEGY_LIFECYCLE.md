# 策略生命週期管理

## 流程總覽

```
上傳回測 CSV ──→ FinLab 五維分析 ──→ 確認存檔（版本化）
                                          │
                                    ┌─────┴─────┐
                                    ▼           ▼
                              策略庫（卡片牆）  景氣適配
                              排名 / 比較     上架 / 下架
                                    │
                                    ▼
                            實單 vs 回測驗證
                            策略是否失效？
```

---

## 階段一：上傳 & 分析

### 輸入
- 回測 CSV / Excel（Python 回測系統產出）
- 策略名稱、初始資金、版本號

### 分析內容
| 維度    | 指標                              | 說明            |
| ----- | ------------------------------- | ------------- |
| 獲利能力  | CAGR、年化報酬、月均報酬                  | 策略賺不賺錢        |
| 抗風險   | MDD、VaR、CVaR、Downside Deviation | 最壞情況能撐多少      |
| 風險報酬比 | Sharpe、Sortino、Calmar           | 每承擔 1 單位風險賺多少 |
| 勝率期望值 | 勝率、盈虧比、期望值、Kelly                | 長期是否正期望值      |
| 交易流動性 | 交易次數、平均持倉天數、換手率                 | 策略是否有足夠樣本     |

### 蒙地卡羅模擬
- 基於歷史交易分布，模擬 1000 條未來路徑
- 產出 P50 / P75 / P95 預期報酬區間
- 判斷策略穩健度

### 產出
- 五維雷達圖
- 綜合評分（0-100）
- 各維度子評分

## 階段二：審核 & 版本化

### 版本規則
```
策略名稱 + 版本號
例：
  S1-動能 ETF 輪動 v1.0    ← 初版回測
  S1-動能 ETF 輪動 v1.1    ← 調參後回測
  S1-動能 ETF 輪動 v2.0    ← 邏輯大改
```

### 存檔位置
- Google Sheets `strategy_performance` 分頁
- 欄位：strategy_id, strategy_name, version, run_date, start_date, end_date, cagr_pct, sharpe, sortino, mdd_pct, calmar, win_rate_pct, trades, avg_profit_pct, avg_loss_pct, ev_pct, kelly_full, kelly_half, file_name, uploaded_at

### 版本對比
- 同一策略不同版本的 KPI 變化趨勢
- v1.0 → v1.1：CAGR 提升？MDD 降低？
- 判斷調參方向是否正確

## 階段三：策略庫

### 策略卡片
每張卡片顯示：
- 策略名稱 + 最新版本
- 五維雷達圖縮圖
- 關鍵 KPI（CAGR、Sharpe、MDD、勝率）
- 策略特性標籤（ETF 輪動 / 動能 / 均值回歸 / ...）
- 狀態燈號

### 策略狀態
| 狀態 | 說明 | 顏色 |
|------|------|------|
| 🟢 上架中 | 正在跑實單 | 綠 |
| 🟡 測試中 | 回測通過、紙上交易驗證中 | 黃 |
| 🔴 下架 | 策略失效或不適合當前市場 | 紅 |
| ⚪ 研究中 | 回測開發階段 | 灰 |

### 策略排名
依綜合評分排序，考量：
1. Sharpe（風險調整報酬）
2. CAGR（絕對報酬）
3. MDD（最壞回撤）
4. 勝率 × 盈虧比（期望值）

### 策略比較
勾選 2-3 個策略，並排對比：
- 雷達圖疊加
- KPI 表格對比
- NAV 曲線重疊

## 階段四：景氣適配

### 景氣四象限
```
        通膨上升
           │
  滯脹     │     過熱
  (防守)   │   (商品/價值)
───────────┼───────────
  衰退     │     復甦
  (債券/現金)│   (成長/動能)
           │
        通膨下降
```

### 每個策略標記適配景氣
| 策略 | 復甦 | 過熱 | 滯脹 | 衰退 |
|------|------|------|------|------|
| 動能 ETF 輪動 | ✅ 強 | ✅ 中 | ❌ 弱 | ❌ 弱 |
| 均值回歸 | ✅ 中 | ✅ 強 | ✅ 中 | ❌ 弱 |
| 台股強勢選股 | ✅ 強 | ✅ 中 | ❌ 弱 | ❌ 弱 |
| 債券配置 | ❌ 弱 | ❌ 弱 | ✅ 中 | ✅ 強 |

### 上架/下架建議
根據當前景氣象限（from `macro_state`）：
- 當前象限適合的策略 → 建議上架
- 不適合的 → 建議下架或減碼
- 狀態轉換時 → 發出預警

## 階段五：實單 vs 回測驗證

### 資料來源
| 資料 | 來源 |
|------|------|
| 回測預期 | `strategy_performance`（CAGR、勝率、MDD 等） |
| 實際交易 | `trades`（已實現損益）+ `broker_positions`（未實現） |

### 驗證指標
| 指標 | 回測 | 實單 | 落差 | 判定 |
|------|------|------|------|------|
| 年化報酬 | +14% | +11% | -3% | 🟢 正常（<5%） |
| 勝率 | 70% | 65% | -5% | 🟡 注意 |
| 最大回撤 | -3.8% | -6.2% | -2.4% | 🟡 注意 |
| 盈虧比 | 2.1 | 1.4 | -0.7 | 🔴 警告 |

### 策略健康度判定
| 等級 | 條件 | 動作 |
|------|------|------|
| 🟢 健康 | 所有指標落差 < 閾值 | 繼續運行 |
| 🟡 注意 | 1-2 項指標偏離 | 觀察，縮小部位 |
| 🔴 失效 | 3+ 項偏離，或 MDD 超過回測 2 倍 | 下架，回去階段一重新分析 |

### 失效後流程
```
策略失效 → 下架 → 回到階段一
  → 檢討原因（市場結構變化？參數過擬合？）
  → 調整策略邏輯或參數
  → 重新上傳回測 → v2.0
  → 通過分析 → 重新上架
```
