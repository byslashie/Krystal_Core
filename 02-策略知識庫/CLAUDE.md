## 📖 知識庫結構

### 策略文檔規則
- 格式：Markdown + YAML front matter
- 命名：`strategy_[策略名]_[版本].md`
- 包含：目標、邏輯、參數、績效、風險

### 策略文檔模板
```markdown
---
name: [策略名稱]
type: [趨勢/均值回歸/套利/混合]
created_date: [YYYY-MM-DD]
author: Krystal
status: [研究/測試/實盤/已棄用]
win_rate: [%]
profit_factor: [數字]
---

## 策略邏輯
[詳細說明]

## 技術指標
- [指標1]
- [指標2]

## 進場條件
1. [條件1]
2. [條件2]

## 出場條件
- 止盈：[%]
- 止損：[%]

## 回測績效
- 勝率：[%]
- 平均獲利：[%]
- 最大回撤：[%]

## 實盤表現
[統計數據]

## 注意事項
[風險和改進方向]
```

## 📊 分析文檔規則

### 市場分析
- **技術面**：均線系統、支撐阻力、K 線型態
- **籌碼面**：外資、投信、大戶持股變化
- **基本面**：業績、產業趨勢、政策影響

### 分析文檔命名
- `analysis_[日期]_[主題].md`
- 例：`analysis_2026-03-20_AI晶片產業.md`

## 🔍 研究方法

### 定量分析
- 使用 Python 進行回測
- 統計學驗證
- 數據驅動的決策

### 定性分析
- 新聞和市場事件追蹤
- 產業研究
- 公司基本面分析

## 📝 文檔管理規則

### 版本控制
- 每次重要更新創建新版本
- 命名：`strategy_[名稱]_v1.0.md`
- 保留歷史版本以追蹤演進

### Git 提交規則
- 新策略：`📚 新策略: [策略名]`
- 策略更新：`📚 策略更新: [策略名]`
- 分析報告：`📊 分析: [主題]`
- 知識整理：`📖 知識: [主題]`
