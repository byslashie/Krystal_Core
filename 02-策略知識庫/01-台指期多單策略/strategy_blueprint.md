# 台指期多單策略知識庫

## 策略中心思想
**多台指期 + 短線日盤 + 高勝率（63%）**

- 交易商品：台指期貨（小型或大型）
- 交易方向：多（看多、買進）
- 持倉時間：30-120 分鐘（極短線）
- 核心邏輯：大盤恐慌訊號 + 高波動 + 逆價差組合進場
- 歷史績效：171 筆交易，勝率 63.2%，累計獲利 NT$553,100（年化 3.3%）

---

## 📊 核心指標體系

### 1. **%B 指標（布林通道）**
```
用途：判斷超買超賣水位
計算：(Close - 下軌) × 100 / (上軌 - 下軌)
設定：Length=20, Range=2
進場區間：%B < 20 且 > -20（中段偏低，準備進場）
出場區間：%B > 72 且 < 120（偏高，準備出場）
```

### 2. **VIX 恐慌指數**
```
用途：市場波動性判斷
閾值：>17（高恐慌，適合空單進場）
來源：getsymbolField("VIX.TF", "收盤價", "5")
應用：條件3 = VIX > 17
```

### 3. **大盤累委買賣（期貨委買委賣成筆差）**
```
用途：主力動能判斷
計算：累賣成筆 - 累買成筆
應用：
  - condition5：連續3根低於前值 + 當根是20根最低 + 小於-1500 → 做空訊號
  - condition8：連續3根高於前值 + 5根平均 > 當根 → 出場訊號
```

### 4. **外資淨買賣比例**
```
用途：機構投資人方向
設定：20 日觀察期
計算：外資買賣超 > 0 的天數 / 20
狀態：未在進場條件中啟用（條件1被註解）
```

### 5. **大盤 EMA（60 日）**
```
用途：長期趨勢判斷
條件2：收盤 < 60EMA × 0.997（大盤在長期均線下方，空單環境）
```

### 6. **價差判斷**
```
正價差：期貨 > 現貨（condition60）
逆價差：期貨 < 現貨（condition6） ← 進場必備
應用：進場時需要逆價差 + 出場時需要正價差（對沖邏輯）
```

### 7. **K線形態**
```
條件11：Close < Open 且 (High-Low)/Close × 100 > 0.13
應用：今根是下跌K + 波動大於 0.13%（高波動下跌，空單訊號）
```

### 8. **成交量**
```
條件10：V > 5根平均成交量
應用：確保有足夠成交量進場
```

## 🚀 進場邏輯

### 進場條件組合（ALL 條件需滿足）
```
if condition2 AND condition3 AND condition6 AND condition11 AND condition10:
    EntryCondition = true
```

| 條件 | 內容 | 含義 |
|---|---|---|
| condition2 | 大盤 < 60EMA × 0.997 | 下跌趨勢 |
| condition3 | VIX > 17 | 高恐慌 |
| condition6 | 逆價差 | 期貨貼水 |
| condition11 | K線下跌 + 高波動 | 下殺訊號 |
| condition10 | 成交量 > 5根均 | 量能確認 |

### 進場執行
```
IF position = 0 AND filled = 0 AND EntryCondition:
    SetPosition(1, addSpread(close, 0), label:="多進場")
    // 注：label 說 "多進場"，但邏輯是空單 → 可能是標籤錯誤
```

## 📤 出場邏輯

### 出場方式 1：高位反轉出場
```
IF condition9 AND condition60:
    SetPosition(0, label:="%B高+正價差")

條件9：%B > 72 AND %B < 120（在高位）
條件60：正價差（期貨升水）
邏輯：高位遇見升水 → 短線高點 → 快速出場
```

### 出場方式 2：委買衝高出場（目前註解）
```
// IF condition5 AND condition6:
//     SetPosition(0, label:="委買衝高")

原意：委買成筆開始增加 + 逆價差消失 → 主力護盤 → 出場
狀態：暫未啟用
```

### 出場方式 3：尾盤強制平倉
```
IF currentTime >= exit_time (預設134000) AND currentDate = EntryDate + 1:
    SetPosition(0, label:="尾盤平倉")

應用：跨日持倉時，次日下午必須平倉
```

### 出場方式 4：停損
```
IF loss_point > 0 AND Close <= FilledAvgPrice - loss_point:
    SetPosition(0, label:="停損")

設定：loss_point = 200（200 點停損）
```

## ⚠️ 已知問題 & 改進方向

### 問題 1：缺少時段限制
- 需求：日盤 08:45-13:25 進場，13:25-15:00 禁止
- 現狀：無時段檢查
- 影響：違反策略規則

### 問題 2：缺少每日進場次數限制
- 需求：每日最多 2 次進場
- 現狀：無計數邏輯
- 影響：無法控制每日交易頻率

### 問題 3：缺少 ATR 波動分化
- 需求：ATR ≥ 25（趨勢）用 Trailing；ATR < 25（盤整）固定 40 點
- 現狀：出場邏輯固定（%B 高位 + 正價差）
- 影響：無法適應不同市況

### 問題 4：缺少週五限制
- 需求：週五全天禁止進場
- 現狀：無周期檢查
- 影響：風險管理不足

### 問題 5：變數命名不規範
- value1, value2, value11, value12, value111...
- 難以維護和理解邏輯

### 問題 6：條件 1（外資淨買賣）被註解
- 原設想可能是判斷盤整
- 目前不在進場邏輯中，邏輯不完整
