## 一、策略本質（一句話）

**「每月第一個交易日，從合格池挑強勢股建底倉；漲 10% 加碼 1 次（最多兩次）；用『成本錨/高點錨/贏家 trailing/鎖利 1/3』四層出場機制收網。」**

## 三、進場流程（單股決策樹）

```
每根 Bar 起始
  │
  ├─ filled = 0？ ──否→ 已在倉，跳到出場流程
  ├─ IsNewMonth？ ──否→ 等下個月
  ├─ isSessionFirstBar 且 isfirstcall("RealBar")？ ──否→ 跳過
  │
  ↓ 五重進場過濾（全部通過才下單）
  ├─ ① IsListedSymbol = true（仍在上市櫃）
  ├─ ② vix(VixLookback) = 0（恐慌指數 NOT 報警）
  ├─ ③ Close > KAMA（日線 KAMA 之上，動能延續）
  ├─ ④ Month ≠ 9（避開 9 月——歷史最差月份）
  ├─ ⑤ W_Trend = 1（週 SuperTrend 多頭）
  └─ ⑥ Rule2_Pass = true（流動性達標）
        │
        ↓ 全過 → 計算底倉口數
        OrderQty = Money / Close
        Multipler = (上一筆是否大賺/小賺/賠錢 → 1.5 / 1.0 / 0.5)
        SetPosition(OrderQty × Multipler, "● 進場")
```

### 過濾條件白話版

| 過濾 | 邏輯 | 為什麼 |
| --- | --- | --- |
| `IsListedSymbol` | 還活著的股票 | 避開下市 |
| `vix(VixLookback) = 0` | 過去 400 天的 VIX 未觸頂 | Momentum crash 防護 |
| `Close > KAMA` | 日線 KAMA(2,30,8) 之上 | 動能濾網 |
| `Month ≠ 9` | 跳過 9 月進場 | 歷史季節因子 |
| `W_Trend = 1` | 週 SuperTrend ATR(22)×3 多頭 | 大週期方向 |
| `Rule2_Pass` | 5 日均量 > 1000 張（台）/ 50 萬股（美） | 流動性 |

## 四、加碼邏輯

```
條件全滿足才加碼：
  ① 已有部位 (filled > 0)
  ② 加碼次數 < 2 (Max_Pyramid_Times)
  ③ 還沒觸發任何部分出場 (PartExited1 = false)
  ④ 當前報酬 ≥ 10% (Pyramid_Threshold)
  ⑤ 大盤未崩 (PausePyramid = false)
        ↑ 規則 1：大盤近 20 日跌 -8%，暫停加碼 15 天

加碼口數 = BaseOrderQty × 0.5 × Rule2_Scale × Rule7_Scale
                       ↑          ↑              ↑
                    固定 50%    流動性懲罰     波動率縮放
                                (低量股 0.5)   (Vol60/Vol20)
```

**關鍵保護**：加碼後若觸發任何**部分出場（如鎖利 1/3）**就不能再加。

## 五、出場優先序（**這是 V4.1 的精華**）

```
每根 Bar 計算：
  ProfitPct = (Close - filledAvgPrice) / filledAvgPrice
  PeakSinceEntry = MaxList(PeakSinceEntry, High)
  MaxRunupPct = (Peak - Cost) / Cost × 100

  CostStopLine = Cost × (1 - Vol20 × 4)     // 成本錨（保本）
  PeakStopLine = Peak × (1 - Vol20 × 6)     // 高點錨（保獲利）

  ┌────────────────────────────────────────────────────┐
  │ 0. 贏家模式切換：浮盈 ≥ 30% → WinnerModeOn = true │  ← 一旦進贏家
  └────────────────────────────────────────────────────┘    雙錨/時間出場
                                                            全部失效，只靠
                                                            trailing 防守
  優先序（從上到下，第一個命中就執行）：

  ① 浮盈 ≥ 35% → 賣 1/3，留 2/3 [鎖利1/3]
       └─ 此筆「永久」不能再加碼，只能等其他出場
  ② WinnerMode = true 且高點回吐 15% → 全出 [贏家回吐]
  ③ 非 WinnerMode 且 Close ≤ MAX(成本錨, 高點錨) → 全出 [動態Vol停損]
  ④ Close ≤ Cost × (1 - 15%) → [停損]（V4.1 預設關閉）
  ⑤ MaxRunupPct ≥ 10% 且 Close ≤ Cost → [保本]（V4.1 預設關閉）
  ⑥ 持有 ≥ 110 天且個股漲幅 < 大盤漲幅 → [時間上限且轉弱出場]
  ⑦ 持有 ≥ 60 天且個股 < 大盤 且 Close < KAMA → [相對弱勢踢除]
  ⑧ 浮盈 ≥ 50% 且 Close cross below KAMA_W → [KAMA大賺出場]
  ⑨ VIX = 1 且 W_Trend = -1 → [VIX+週壓力]（底線，贏家也適用）

  並行的部分出場：
  F.1) 非 WinnerMode 且 MaxRunup ≥ 10% 且 Close ≤ Peak × (1 - Vol20 × 6)
       → 賣 50% [動態Vol減碼50%]
```

### 出場原因實盤分布（V4.1，184 筆）

| 出場原因 | 次數 | 占比 | 平均報酬 | 角色 |
| --- | --- | --- | --- | --- |
| 贏家回吐 | 85 | 46% | **+53.6%** | 主力獲利出場 |
| 動態Vol停損 | 70 | 38% | -9.3% | 停損閘門 |
| 鎖利1/3 | 26 | 14% | +45.2% | 部分鎖利（不全出） |
| VIX+週壓力 | 3 | 2% | -24.5% | 底線防護 |

**核心觀察**：贏家回吐 + 鎖利 1/3 = 60% 的出場原因，平均報酬都 +45% 以上 → 這套出場機制**真的有抓到大魚**。

## 七、V4.1 vs V4 vs 每月進場 — 為什麼 V4.1 勝出

實盤數據（2020-01 ~ 2026-03 同期）：

| 指標 | 每月進場 | V4 | **V4.1** |
| --- | --- | --- | --- |
| CAGR | 189% | 496% | **692%** |
| MDD | -14.5% | -46.2% | -32.9% |
| 夏普 | 1.54 | 0.80 | **2.48** |
| 勝月比 | 65% | 55% | **68%** |
| 勝率 | 66% | 63% | **68%** |
| 中位數單筆% | 7.9% | 7.4% | **21.2%** |

**V4.1 的決勝關鍵**（從 V4 → V4.1）：
1. **雙錨停損取代相對弱勢踢除為主防線** → 防守反應時間從 60 天縮到 25 天
2. **贏家 trailing 取代固定時間出場** → 抓到華城 +149%、緯創 +204% 那類大波段
3. **浮盈鎖利 1/3** → 在 +35% 強制鎖一部分，**避免大贏變沒贏**
4. **平均持有從 90 日縮到 31 日** → 資金週轉率提升 3 倍
