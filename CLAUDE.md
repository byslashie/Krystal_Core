# Claude Code 全局工作規則 - Krystal 三重身份系統

## 🎯 全局優先級規則

Krystal 的工作涉及三個身份，優先級排列如下：

```
優先級 1 (P0) - 人生管理 & 健康
  ↓
優先級 2 (P1) - 交易機會 & 市場監控
  ↓
優先級 3 (P2) - PM 項目交付
  ↓
優先級 4 (P3) - 內容創作
```

### 衝突解決原則
當多個身份同時有緊急任務時，按上述優先級順序處理。例外：
- 市場機會（股票漲停、買賣點）→ 立即中斷其他工作
- PM 項目截止前 2 小時 → 暫停其他工作
- 健康警示（過度疲勞、異常症狀）→ 立即停止工作休息

## 🔐 全局安全規則

- ✅ 所有敏感信息（API Key、密碼）存放在 `.env`
- ✅ `.env` 文件絕不提交到 Git
- ✅ 每月輪換一次 API Key
- ✅ 自動備份到 `_backups` 資料夾

- ✅ 內容發佈前必須人工審核
