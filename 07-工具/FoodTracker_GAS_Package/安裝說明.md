# 🥗 FoodTracker — Google Apps Script 版 安裝說明

> 這個 App 完全免費、不需要任何伺服器，所有資料存在你自己的 Google Sheets 雲端帳號裡。
> 只要有 Google 帳號，5 分鐘內即可完成安裝，加到手機主畫面後可像 App 一樣使用。

---

## 📋 安裝前準備

你需要：
- ✅ **Google 帳號**（一定要有）
- ✅ **Gemini API Key**（免費申請，步驟見下方）
- ✅ 手機或電腦連接網路

---

## 🔑 申請 Gemini API Key（免費）

1. 開啟瀏覽器，前往 [https://aistudio.google.com](https://aistudio.google.com)
2. 用你的 Google 帳號登入
3. 點右上角 **「Get API Key」**
4. 點 **「Create API key」** → 選一個專案（或新增專案）
5. 複製 API Key → 先貼到記事本備用

> 📌 **API Key 看起來像這樣：** `AIzaSyXXXXXXXXXXXXXXXXXXXXX`

---

## 🗂️ 步驟 1：建立 Google Sheets（資料庫）

1. 瀏覽器開啟 [https://sheets.new](https://sheets.new)（會自動新增空白試算表）
2. 左上角試算表名稱，改成：**`FoodTracker DB`**
3. 複製網址列的 **Spreadsheet ID**：
   - 網址格式：`https://docs.google.com/spreadsheets/d/【這段就是ID】/edit`
   - 把 `/d/` 和 `/edit` 之間那段複製起來貼到記事本備用（大約 40-50 個字元）

---

## 📁 步驟 2：建立 Google Drive 食物圖片資料夾

1. 開啟 [https://drive.google.com](https://drive.google.com)
2. 點左上角 **「新增」→「資料夾」**
3. 命名為 **`FoodPhotos`** → 點「建立」
4. 點進那個資料夾，複製網址列最後的 **Folder ID**：
   - 網址格式：`https://drive.google.com/drive/folders/【這段就是ID】`
   - 複製最後那段（大約 33 個字元）貼到記事本備用

---

## 💻 步驟 3：建立 Google Apps Script 專案

1. 開啟 [https://script.google.com](https://script.google.com)
2. 點 **「新專案」**
3. 左上角專案名稱改為：**`FoodTracker`**

### 貼入後端程式碼（Code.gs）

4. 點左側 **「Code.gs」**
5. 選取編輯器裡所有內容（Ctrl+A），**全部刪除**
6. 打開本包裡的 **`Code.gs`** 檔案，全選複製
7. 貼到 Apps Script 編輯器裡
8. 按 **「儲存」**（Ctrl+S 或磁碟片圖示）

### 新增前端頁面（Index.html）

9. 點編輯器上方的 **「+」** → 選 **「HTML」**
10. 命名為 **`Index`**（注意：**不要加 `.html`**，就打 Index）
11. 打開本包裡的 **`Index.html`** 檔案，全選複製
12. 貼到 Apps Script 的 Index.html 編輯器裡
13. 按 **「儲存」**（Ctrl+S）

---

## ⚙️ 步驟 4：設定三組 API 金鑰

1. 左上角點 **齒輪圖示「專案設定」**
2. 往下滾，找到 **「指令碼屬性」** 區塊
3. 點 **「新增屬性」**，依序加入以下三組：

| 屬性名稱（完全照抄）| 值（你的 ID）         |
|-------------------|----------------------|
| `SPREADSHEET_ID`  | 步驟 1 複製的 Sheets ID |
| `GEMINI_API_KEY`  | 申請的 Gemini API Key  |
| `DRIVE_FOLDER_ID` | 步驟 2 複製的 Drive 資料夾 ID |

> ⚠️ **注意：屬性名稱必須完全一樣，大小寫不能錯！**

4. 點 **「儲存指令碼屬性」**

---

## 🚀 步驟 5：部署為網頁應用程式

1. 點右上角 **「部署」→「新增部署作業」**
2. 左上角類型選 **「網頁應用程式」**
3. 設定如下：
   - **說明**：`v1`
   - **執行身分**：`我（你的 Google 帳戶）`
   - **有權存取的使用者**：`所有人（Anyone）`
4. 點 **「部署」**
5. 出現**授權視窗**→ 點「授權存取」→ 選你的 Google 帳號 → 點「允許」
6. 複製最後出現的 **網頁應用程式網址**（長網址，開頭像 `https://script.google.com/macros/s/...`）

---

## 📱 步驟 6：加到手機主畫面（可像 App 使用）

### iPhone / iPad（Safari）：
1. 用 **Safari** 開啟步驟 5 複製的網址
2. 點下方分享按鈕（方框+箭頭圖示）
3. 選 **「加入主畫面」** → 點「新增」
4. 主畫面會出現 FoodTracker 圖示 🎉

### Android（Chrome）：
1. 用 **Chrome** 開啟網址
2. 點右上角三點選單
3. 選 **「加入主畫面」** 或 **「安裝應用程式」**
4. 點「新增」

## ❓ 常見問題

**Q：部署後出現「此應用程式未經驗證」？**
A：點「進階」→「前往 FoodTracker（不安全）」→ 允許。這是 Google 對個人開發的 Script 的正常警告，你自己的程式碼是安全的。

**Q：圖片辨識失敗？**
A：確認 `GEMINI_API_KEY` 有正確填入，且 API Key 未過期。

**Q：儲存資料後刷新沒有看到？**
A：GAS 第一次執行較慢，等 5~10 秒後重整頁面。

**Q：我的資料存在哪裡？**
A：所有飲食/訓練資料都存在你 Google 帳號的 Sheets（步驟 1 建立的那個），完全在你自己的 Google 雲端，開發者看不到。
