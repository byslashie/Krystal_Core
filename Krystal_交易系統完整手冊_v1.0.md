# Krystal 交易系統完整手冊 v1.0

> **核心流程：** 原則（理念）→ 規則（具體條件）→ 資料（市場數據）→ 判斷（符合與否）→ 執行（配置建議/交易）

## 系統架構總覽

```
【感知層】
  M1（總經） + M2（季節性） + N3（技術/情緒）+ Panopticon（地緣情報）
         ↓
【決策層】
  D1（決策引擎）→ A1（資金配置器）
         ↓
【策略層】
  S1（美股ETF輪動）/ S2（台股ETF輪動）/ S3（台股強勢股）
  S4（期貨日內）/ S5（空頭對沖）/ S6（可轉債）
         ↓
【風控層】
  R1（規則風控）→ R2（監控警報）
         ↓
【執行層】
  EXE（元大/IB/Schwab）→ fills.csv
         ↓
【監控層】
  T1（KPI通道圖）→ N1（通知備份）→ 回授 D1
```

## 第零章：交易原則（最高層理念）

### 0.1 系統型交易者宣言

| # | 原則內容 |
|---|---------|
| 1 | 市場循環不可避免，策略必須跟著市場環境調整 |
| 2 | 短線可因「預期/消息」布局；長線須確認「現實改善」才建倉 |
| 3 | 長期持倉必須順趨勢，絕不逆勢操作 |
| 4 | 資金流向預示下一步方向，建立系統化方法追蹤資金流 |
| 5 | 情緒極端時（狂熱/恐慌）可短期逆向操作，捕捉修正/反彈 |
| 6 | 趨勢明確時積極交易，盤整混亂時降低出手頻率 |
| 7 | 資產配置依市場環境而定，絕不因情緒衝動亂改持倉 |
| 8 | 以控制風險為前提，追求穩健且超額的長期報酬 |
| 9 | 定期回顧和優化規則，不盲目堅持無效方法，但要有紀律執行每次決策 |
| 10 | 每次交易決策必須以數據/模型/規則為依據，不憑感覺或直覺 |

### 0.2 交易守則（執行紀律）

| # | 守則內容 |
|---|---------|
| 1 | 絕不交易未經回測的商品 |
| 2 | 嚴格遵守策略流程進出場 |
| 3 | 絕不貪最後一筆，見好就收 |
| 4 | 只有判斷明確符合設定條件時才進場 |
| 5 | 錯誤要認，虧損要勇敢認賠出場 |
| 6 | 單筆虧損不超過總資金 3%，理想控制在 1–1.5% |
| 7 | 獲利單要讓獲利奔馳，不過早出場 |
| 8 | 做好備案（下單+套保），應對突發狀況 |
| 9 | 開盤前完成 Checklist，確認當天盤勢 |
| 10 | 趨勢持續時耐心持有或追加部位 |
| 11 | 不因表面漲跌影響交易邏輯，堅持策略不隨便改變計畫 |

### 0.3 資金哲學

- **70/30 法則（S3 強勢股）：** 70% 投入執行 + 30% 戰略緩衝金（絕不動用）
- **單筆風險上限：** ≤ 1–2%（全系統守則）
- **每月最大回撤門檻：** 8–12%
- **連虧保護：** 連虧 3 次 → 強制停手 2 天

### 0.4 人格適配性

- **最適合：** 趨勢型 × 系統型 × 模型型（月調/波段，非高頻當沖）
- **最不適合：** 逆勢攤平、情緒操盤、抄底、不執行停損
- **行動原則：** 系統信號 > 盤感猜測

## 第一章：市場環境判斷層（Classifiers）

> **目標：** 在執行任何交易前，先確認目前所在的市場環境象限。環境判斷是所有策略的「開關」。

### 1.1 M1 總經分類器（經濟週期）

**更新頻率：** 每月（新數據公布後）
**產出：** `macro_state.csv`（date, regime, regime_score）

#### 1.1.1 原則
以 GDP/PMI/CPI/失業率判斷景氣四象限。景氣象限決定策略組合與資金比重。

#### 1.1.2 分類條件

| 象限 | GDP YoY | PMI | CPI | 失業率 | 殖利率曲線 | 對應配置 |
|------|---------|-----|-----|--------|----------|---------|
| **Growth（擴張）** | 2–4% | >52 持續 | 穩定 | 低且穩定 | 正斜率 | S1/S3 全開，偏多 60–70% |
| **Recovery（復甦）** | 低位回升 >0% | <50→升破50 | 溫和上升 | 見頂下滑 | 逐漸陡化 | S1/S3 降權，偏多 50% |
| **Slow Growth（放緩）** | 下降但 >0% | 跌回 <50 | 快速下降 | 轉升 | 平坦或倒掛 | S5 啟用，中性 30–40% |
| **Recession（衰退）** | <0% | <45 | 接近 0 | 快速上升 | 倒掛 | S5/S6 主力，防守 20–30% |

#### 1.1.3 加權打分公式

```
regime_score = PMI_zscore × 0.35
             + GDP_zscore × 0.25
             + CPI_zscore × 0.20
             + 失業率_zscore × (-0.20)

分類門檻：
  score > 1.0 → Growth
  score > 0.0 → Recovery
  score > -1.0 → Slow Growth
  else → Recession
```

#### 1.1.4 資料來源

| 指標 | 來源 |
|------|------|
| GDP YoY | FRED (`GDPC1`) |
| PMI | Institute for Supply Management |
| CPI YoY | FRED (`CPIAUCSL`) |
| 失業率 | Bureau of Labor Statistics (`UNRATE`) |
| 10Y–2Y 殖利率差 | FRED (`T10Y2Y`) |

### 1.2 M2 季節性規則（時間週期）

**更新頻率：** 月更 + 事件觸發
**產出：** `seasonality_flags.csv`（date, month_tag, TOM, earnings_season, fomc_week, options_expiry）

#### 1.2.1 原則
特定時間點市場流動性與波動率有系統性規律，需主動調整槓桿與入場頻率。

#### 1.2.2 事件旗標規則

| 旗標 | 觸發條件 | 系統反應 |
|------|---------|---------|
| **TOM（月轉效應）** | 每月首 5 個交易日 | S1 可加倉，趨勢追蹤加強 |
| **FOMC 週** | 聯準會公告週 | 全系統降槓桿，不新建倉 |
| **財報季** | Q1/Q2/Q3/Q4 密集期 | S3 謹慎進場，財報日前規避留倉 |
| **結算日** | 台指/選擇權每月結算 | S4 暫停或降規模 |
| **12月效應（Santa Rally）** | 12月後半 | 可加碼趨勢 |
| **9月效應（弱季）** | 9月全月 | 全面降低曝險 |

#### 1.2.3 年度月份策略矩陣

| 月份 | 市場特性 | 主軸策略 | 倉位方向 | 特別注意 |
|------|---------|---------|---------|---------|
| 1月 | 新年效應、小型股強 | S1 科技/成長 ETF | 偏多 60–70% | 小型股配置 10–20% |
| 2月 | 財報季 | S6 因子選股（EPS+動能） | 偏多 50% | 財報大波動→單筆降至 1% |
| 3月 | Q1 收官 / TOM | S1 + 月初效應 | 中性 40–50% | TOM 結束後減碼 |
| 4月 | 歷史偏強 | S3 強勢股 + S1 | 偏多 ≥60% | 高檔設移動停利 |
| 5月 | Sell in May | 防禦 ETF（高股息/低波） | 降低 30–40% | 反向ETF預案 |
| 6月 | 夏季量縮 | 均值回歸 / cbAS 套利 | 中性 30–40% | 降低強勢股曝險 |
| 7月 | 科技財報旺季 | S3 + QQQ/SOXX | 偏多 60% | 財報日不留倉 |
| 8月 | 夏末疲弱 | S1 防守版 + 均值回歸 | 中性 30–40% | 波動升→單筆 0.8% |
| 9月 | 全年最弱 | 短債/黃金 / cbAS | 防守 20–30% | 觸發 -2σ 全面降權 |
| 10月 | 高波動 | 均值回歸 + VIX 避險 | ≤30% | 反衝過大立即停利 |
| 11月 | 冬季行情啟動 | S1（成長/動能）+ S3 | 偏多 60% | 贏家加碼、落後汰換 |
| 12月 | Santa Rally | S3 + S1 + TOM | 偏多 70% | 移動停利鎖利跨年 |

### 1.3 N3 大盤趨勢判定器（技術/情緒）

**更新頻率：** 每日
**產出：** `market_trend_state.csv`（date, index, score, state, confidence）

#### 1.3.1 原則
M1 是月維度的環境判斷，N3 補充「當下」的市場方向，避免在空頭盤做多、在多頭盤做空。

#### 1.3.2 判定條件

| 指標 | 條件 | 對應狀態 |
|------|------|---------|
| 大盤 vs SMA200 | 指數 > 200日均線且均線上彎 | Bull（多頭） |
| 大盤 vs SMA200 | 指數 < 200日均線且成交量放大下跌 | Bear（空頭） |
| ADX | ADX < 20 且價在均線附近反覆 | Neutral（盤整） |
| VIX | VIX > 30 | Bear 或避險 |
| DXY | DXY > 105 且資金流出風險資產 | 避險 |
| MACD | 黃金交叉 / 死亡交叉 | 輔助確認 |

#### 1.3.3 情緒指標補充

| 指標 | 條件 | 解讀 |
|------|------|------|
| CNN Fear & Greed | >80 = 貪婪 / <20 = 恐慌 | 極端時可考慮反向 |
| VIX | >30 恐慌 / <15 極度安逸 | 趨勢延續或反轉風險 |
| Put/Call Ratio | >1.2 恐慌 / <0.7 太樂觀 | 轉折風險 |

### 2.1 S1 美股 ETF 輪動（核心穩定複利）

#### 2.1.1 原則
用長期相對動能篩選最強美股 ETF，月調一次。目標：CAGR ≥15%，MDD < -20%。

#### 2.1.2 規則

**進場：**
- 標的池：QQQ、SPY、ACWI 等主要美股 ETF
- 動能分數：12 個月滾動報酬排名（R12M）
- 月初選出排名第一的 ETF，滿倉持有

**出場：**
- 月初排名跌出第一名 → 月初換倉至新冠軍

#### 2.1.3 啟用條件（D1 判斷）

| 條件 | 要求 |
|------|------|
| M1 象限 | Growth 或 Recovery（非 Recession） |
| N3 狀態 | Bull 或 Neutral（非 Bear） |
| M2 事件 | 非 FOMC 週（事件週不新建倉） |
| D1 狀態 | 明確列為 active |

#### 2.1.4 回測績效（2019–2025）

| MDD | -14% |

### 2.2 S2 台股 ETF 輪動

#### 2.2.1 原則
台股版動能輪動，利用 0050 / 00631L（正2）的相對強弱切換，捕捉台股多頭加速段。

#### 2.2.2 規則

**選股：**
- 標的池：0050 + 00631L（正2）
- 動能評分：R(200天)
- 條件：相對動能 > 0 且大盤週 EMA(60) 通過

**進場：** `收盤 > EMA(週收盤, 60) × 0.997`

**出場：** `收盤 < EMA(週收盤, 60) × 0.997` 或不在標的池

### 2.3 S3 台股強勢股突破（主力進攻策略）

#### 2.3.1 原則
「強者恆強」：鎖定外資/法人買超的強勢股，在突破時順勢進場，讓獲利奔馳。最大資金上限 70%（30% 為不可動用緩衝）。

#### 2.3.2 進場規則

| 條件 | 說明 |
|------|------|
| 時間 | 月初（跨月當日）執行 |
| 標的 | 在 AI 選股清單（法人買超 + 技術突破 + 打分排名） |
| VIX 濾網 | VIX 過高時停止進場（vix=0 才可進場） |
| 趨勢濾網 | 股價站上 KAMA（多頭趨勢確認） |
| 每檔金額 | 100,000 TWD（零股方式） |
| 同時持倉上限 | 最多 5 檔；第 6 個訊號忽略 |

#### 2.3.3 出場規則（六大衛兵機制）

| 衛兵 | 名稱 | 觸發條件 |
|------|------|---------|
| 1 | **初始停損** | 跌破進場價 -8% |
| 2 | **保本（BE）** | 達 +30% 後，若回落至進場價出場 |
| 3 | **第一段停利** | 達 +70%，從高點回落 20% 出場 |
| 4 | **第二段停利** | 達 +100%，從高點回落 30% 出場 |
| 5 | **時間停損** | 持有超過 N 天未達標報酬（大贏單除外）|
| 6 | **清單移除** | 跨月後不在選股名單 |
| 7 | **VIX 緊急出場** | VIX 風險旗標（vix=1）→ 即時全數出清 |

#### 2.3.4 加碼機制

- 條件：出現「獲利加碼」訊號（強勢突破延伸）
- 每次加碼金額：100,000 TWD
- 單一股票加碼上限：2 次（單股最多 300,000 TWD）

#### 2.3.5 執行時間

| 時段 | 動作 |
|------|------|
| 收盤後 14:00 | 計算隔日訊號、計算股數 |
| 次日 09:00–13:25 | 零股執行（市價或限價） |
| 賣出訊號 | 設定次日預約單（勿臨時改單） |

| 最深回撤恢復 | 歷史約 33 天 | — |

### 2.4 S4 期貨日內（不留倉）

#### 2.4.1 原則
日內趨勢跟隨，絕不過夜持倉。收盤必平，保護資金不被隔夜風險侵蝕。

#### 2.4.2 規則

**進場（開盤 15 分鐘 Range）：**
- 觀察開盤後 15 分鐘確認高低區間
- 突破高點 → 做多；跌破低點 → 做空
- 假突破確認後 → 反向（失敗反轉策略）

**停損停利：**

| 類型 | 條件 |
|------|------|
| 單筆停損 | -0.5%（帳戶） |
| 日累計虧損 | -1% → 當日停止交易 |
| 強制平倉 | 收盤前 10 分鐘全平 |

#### 2.4.3 啟用條件

| 條件 | 要求 |
|------|------|
| N3 狀態 | Bull 或 Neutral（有方向性，非盤整洗價日） |
| M2 事件 | 非 FOMC/結算日（自動停用） |
| 連虧限制 | 連虧 3 次 → 停手 2 天 |

### 2.5 S5 空頭盤整 / 對沖

#### 2.5.1 原則
不在空頭市場裸多。N3 = Bear 時啟用，是系統的防守盾牌，保護多頭策略的累積獲利。

#### 2.5.2 規則

| 類型 | 操作 | 說明 |
|------|------|------|
| 高檔盤整反轉 | 假突破後放量黑K，布林收斂+RSI背離→放空 | 技術反轉訊號 |
| 弱勢股放空 | 跌破均線+成交量萎縮+反彈無力→放空 | 選弱出擊 |
| 假突破放空 | 突破高點失敗、隔日大跌 | 誘多失敗反轉 |
| 反向 ETF 對沖 | 買入 SQQQ / 反向0050 ETF | 簡單對沖工具 |
| 區間套利 | 盤整高低點來回，設區間上下止盈止損 | 低方向性操作 |

#### 2.5.3 啟用條件

- N3 = Bear：D1 自動觸發 S5
- M1 = Slow Growth 或 Recession：S5 加大比重
- T1 偏離 ≤ -2σ 連續 N 日：降權其他策略，啟動 S5

#### 2.5.4 停用條件

- N3 翻回 Bull：快速止損空單，轉為跟多
- M1 回到 Growth：S5 降權或停用

## 第三章：風控管理（R1 / R2）

### 3.1 R1 風控規則（規則層）

**產出：** `risk_rules.yaml`, `risk_checks.csv`
**更新：** 即時（每筆訂單前檢核）

#### 3.1.1 原則
損失是系統設計的一部分，受控的損失不是失敗。所有停損在進場前已定義，不在盤中臨時修改。

#### 3.1.2 倉位計算

```
建議倉位 = k × (策略信號強度 / 當前波動率)
k = 係數（依系統整體曝險水位調整）
```

#### 3.1.3 停損族

| 停損類型 | 觸發條件 | 適用策略 |
|---------|---------|---------|
| 固定比例停損 | 跌破進場價 -X% | S3(-8%), S1/S2(-20%) |
| ATR 動態停損 | 跌破 N×ATR | S3 趨勢跟隨段 |
| 鎖利（移動停利） | 從高點回落 Y% | S3 分段停利（衛兵 3/4） |
| 保本停損（BE） | 達 +30% 後回至進場價 | S3 衛兵 2 |
| 時間停損 | 持有 N 日未達目標報酬 | S3 衛兵 5 |

#### 3.1.4 風控硬性規定

| 規則 | 數值 |
|------|------|
| 單筆風險 | ≤ 1–2%（全帳戶） |
| 單日虧損上限 | ≤ 3%（全組合）/ -1%（S4 日內）|
| 策略連虧 | ≥ 3 次 → 冷卻 2 交易日 |
| 組合 MDD 警戒 | 10%（降權）/ 15%（停用高波動模組）|
| S3 帳戶干預線 | -20% → 暫停並檢討 |
| 每日總曝險 | ≤ 50%（期貨另行日額度） |

## 第七章：監控與通知（T1 / N1）

### 7.1 T1 KPI 儀表板 + 通道圖

**產出：** `kpi_dashboard.md`, `sigma_events.csv`, `monthly_report.pdf`
**更新：** 週更 KPI / 即時偏離警報 / 月報

#### 7.1.1 原則
績效曲線在 ±1σ 通道內屬正常；突破 ±2σ 為異常事件，需審查策略。通道圖是「策略是否健康」的長期健診工具。

#### 7.1.2 σ 通道計算

```
通道 = 以 250 日滾動均值 ± N 個標準差
±1σ：正常運作區間
±2σ：異常，觸發審查
連續 2 個月超出 -2σ → 策略暫停/停用
```

#### 7.1.3 核心 KPI 指標

| 指標 | 說明 | 目標 |
|------|------|------|
| CAGR | 年化報酬率（滾動 12M）| > 20% |
| Sharpe | 風險調整報酬 | > 1.2 |
| Sortino | 下行風險調整報酬 | > 1.5 |
| MDD | 最大回撤 | < -15% |
| 勝率 | 獲利筆數比例 | > 55% |
| 盈虧比 | 平均獲利 / 平均虧損 | > 1.5 |
| EV% | 每筆期望值 | > 0% |
| VaR/CVaR | 95%/99% 單日最大損失估算 | — |
