## 🔄 身份切換規則

### ✅ 允許的切換

```
交易 (09:00-12:00) → 午餐 → PM (13:00-17:00)
  理由：有 1 小時的物理分隔和進食時間，足以重置上下文

PM (13:00-17:00) → 晚餐 + 運動 → 內容 (19:30-21:00)
  理由：有 2.5 小時的物理活動和放鬆，足以重置上下文

內容 (19:30-21:00) → 日誌 (22:00-23:00)
  理由：日誌是「反思」而非「新工作」，可以銜接
```

### ❌ 禁止的切換

```
❌ 09:00-12:00 交易時段中途切去做 PM
   理由：進場訊號稍縱即逝，中途切換會錯過機會

❌ 13:00-17:00 PM 深度工作中途切去看市場
   理由：注意力殘留，做 4 小時效果會變成做 8 小時的疲勞

❌ 19:30-21:00 內容創作時關注交易通知
   理由：創作需要心流狀態，通知會破壞流程

❌ 同時開兩個 AI 會話在身份間切換
   理由：這就是「注意力殘留」的定義
```

## 🚨 衝突解決優先級

### 情境 1：交易訊號 vs PM 會議
```
09:00-12:00：交易時段 → 交易優先
13:00-17:00：PM 時段 → PM 優先

交易市場 14:00 有訊號 → 暫停 PM，執行交易
PM 會議 10:30 → 擊延至 12:00 後
```

### 情境 2：交易虧損 vs 心態
```
連虧 3 次 → 停手 (連虧停手規則)
此時：取消 PM 會議？NO
做法：執行日誌反思，隔日恢復常規時程
```

### 情境 3：自媒體機會 vs PM 截止
```
重要內容素材（TA 熱點）→ 記錄，不實時創作
PM 截止前 2 小時 → 暫停自媒，全力 PM
```

### 情境 4：健康警示 vs 所有身份
```
過度疲勞、異常症狀 → 立即停止工作，休息
優先級：P0 健康 > 一切

做法：縮短該日所有時段，增加運動和睡眠
```
