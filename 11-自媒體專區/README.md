## 🤖 自動化系統（核心）

### 日記 → Threads 草稿流程

**一句話：** 你寫日記 → AI 自動萃取洞察 → 生成 3 篇 Threads 草稿 → 你選一篇發出去

### 使用方式

#### 1️⃣ 確認環境設定

```bash
# 確認根目錄有 .env 檔案，且包含：
ANTHROPIC_API_KEY=sk-ant-...
```

#### 2️⃣ 安裝套件

```bash
pip install -r requirements.txt
```

#### 3️⃣ 執行腳本

```bash
# 處理最新日記
cd /Users/chenyu/Desktop/Projects/Krystal_Core
python 07-工具/scripts/journal_to_threads.py

# 或指定日期
python 07-工具/scripts/journal_to_threads.py --date 2026-04-14

# 或批量本週
python 07-工具/scripts/journal_to_threads.py --batch weekly
```

#### 4️⃣ 審核草稿

腳本會生成檔案在：
```
11-自媒體專區/Threads/drafts/YYYY-MM-DD_threads_drafts.md
```

打開檔案，你會看到：
- 核心洞察
- 3 篇不同風格的草稿
- 推薦標籤

#### 5️⃣ 發佈

1. 選一篇你最喜歡的
2. 複製內容到 Threads
3. 貼上標籤
4. 發佈！
5. 將 `drafts/` 檔案移到 `published/` 存檔

## 📝 怎麼寫日記才能生成更好的草稿

### 好的日記特徵

✅ **有核心覺察**
- 「我今天發現...」
- 「我才明白...」
- 「我意識到...」

✅ **有具體例子**
- 不是「人生很複雜」
- 而是「我在 X 件事上發現了 Y 模式」

✅ **有反思層次**
- 不只描述（發生了什麼）
- 還要分析（為什麼）
- 更要思考（現在怎麼辦）

### 不好的日記

❌ **純吐槽** — AI 無法萃取有用的洞察
❌ **碎片化** — 東一句西一句，沒有主軸
❌ **全是隱私** — AI 會自動過濾，但最好自己注意邊界

## 🆘 常見問題

### Q：腳本出錯怎麼辦？

最常見的問題：
1. **ANTHROPIC_API_KEY 沒設定** → 檢查根目錄 `.env` 檔案
2. **anthropic 套件沒裝** → 執行 `pip install anthropic`
3. **找不到日記** → 檢查日記是否在 `04-個人特質與規劃/日記/` 資料夾

### Q：生成的草稿質量很差怎麼辦？

可能的原因：
- 日記內容太散（見上面「怎麼寫日記」章節）
- AI 沒有充分的背景（可以在日記開頭加上「背景」段落）
- API 請求出問題（重試一次）

解決方法：
```bash
# 重新執行
python 07-工具/scripts/journal_to_threads.py --date 2026-04-14
```

### Q：我想要修改生成的內容怎麼辦？

完全可以！草稿只是起點，你可以：
1. 修改內容
2. 改標題
3. 改句子結構
4. 加入個人化觸感

重點是**你覺得真實**。

### Q：能否自動發佈到 Threads？

現階段不行（Threads API 還不成熟）。但流程很快：
1. 複製內容（5 秒）
2. 貼上到 Threads（10 秒）
3. 加標籤（5 秒）
4. 發佈（2 秒）

總共 1 分鐘內。

## 💡 小祕訣

### 讓 AI 生成更好草稿的秘訣

在日記裡加上這些：
- **`[INSIGHT]`**：標記你認為值得分享的洞察
- **`[STORY]`**：具體的例子或經歷
- **`[LESSON]`**：你學到的東西

例如：
```markdown
# 日記 2026-04-18

[INSIGHT] 我發現高共感女性容易把「別人的情緒」當成「自己的責任」。

[STORY] 今天男友又因為工作心情不好，我第一反應是「我要怎麼安撫他」，
而不是「他需要什麼」。

[LESSON] 邊界設定的第一步，是區分「我的問題」和「別人的問題」。
```

這樣 AI 會更精準地萃取。
