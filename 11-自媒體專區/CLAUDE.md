## 🤖 自動化工作流

### 日記 → Threads 流程

```
1. 每天寫日記（04-個人特質與規劃/日記/）
   ↓
2. 每週運行自動化腳本：
   python 07-工具/scripts/journal_to_threads.py
   ↓
3. 系統生成 3 種風格草稿
   - 故事型：個人覺察
   - 價值型：可操作的觀點
   - 提問型：引發互動
   ↓
4. 草稿存入 11-自媒體專區/Threads/drafts/
   ↓
5. 你審核 → 選擇要發的 → 複製到 Threads 發佈
   ↓
6. 已發布文章備份到 11-自媒體專區/Threads/published/
```

### 運行腳本指令
```bash
# 處理最新日記
python journal_to_threads.py

# 指定日期
python journal_to_threads.py --date 2026-04-14

# 批量處理本週日記
python journal_to_threads.py --batch weekly
```

## 📝 內容創作指南

### Threads 文體特徵
✅ 要這樣做：
- 「我發現...」、「有個想法...」開頭
- 一句話一個想法，用短句
- 有停頓感，不是長段落
- 邀請互動：「你呢？」、「有共鳴嗎？」
- 個人化、誠實、不教訓人

❌ 不要這樣做：
- 純粹的技術貼（那是部落格的事）
- 勵志雞湯（虛偽）
- 試圖讓所有人都喜歡
- 過度包裝、不誠實

### 部落格文體特徵
✅ 要這樣做：
- 故事開頭（為什麼你應該讀這篇）
- 系統化論述（邏輯清晰）
- 實戰案例或個人經驗
- 可操作的建議
- 結尾有反思問句

### 選材標準
✅ 可以分享的：
- 你已解決的問題
- 從失敗中學到的東西
- 系統化的思考框架
- 通用的心理洞察

❌ 不分享的：
- 男友的隱私細節
- 家人的具體信息
- 未公開的交易數據
- 其他人的秘密
